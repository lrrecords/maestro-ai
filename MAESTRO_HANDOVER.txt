7. CONFIGURATION

AUTOPILOT CONFIG (data/autopilot_config.json):

Current Settings:
{
  "enabled": true,
  "check_interval_minutes": 60,
  
  "daily_briefing": {
    "enabled": true,
    "time": "09:00",
    "send_email": true
  },
  
  "auto_checkins": {
    "enabled": true,
    "health_threshold": 60,           ← Artists below this trigger check-ins
    "days_since_contact_threshold": 3, ← Must be X days since contact
    "require_approval": true,          ← SAFETY: Human approval required
    "max_per_day": 3                   ← Limit check-ins per day
  },
  
  "critical_alerts": {
    "enabled": true,
    "health_threshold": 40,
    "send_email": true
  },
  
  "rate_limits": {
    "max_emails_per_hour": 5,
    "max_emails_per_day": 20
  }
}

- health_threshold: 50 (more conservative)

- max_per_day: 5

8. HOW TO USE EVERYTHING
================================================================================

COMMAND CENTER:
  python scripts\command_center.py
  
  Options:
  [1] Deep intelligence analysis on any artist
  [2] Generate check-in message (AI-crafted)
  [3] Refresh dashboard
  [4] Full BRIDGE briefing
  [5] Detailed health breakdown
  [6] View learning insights

AUTOPILOT:
  Start:    python scripts\maestro_autopilot.py run
  Queue:    python scripts\maestro_autopilot.py queue
  Test:     python scripts\maestro_autopilot.py test
  Briefing: python scripts\maestro_autopilot.py briefing
  
  Stop: Ctrl+C in the terminal

QUEUE MANAGEMENT:
  View queue:           python scripts\maestro_autopilot.py queue
  Approve all:          Type 'A' in queue interface
  Reject all:           Type 'R' in queue interface
  Approve specific:     Type number (e.g., '1')
  Clear queue:          del data\autopilot_queue.json

ARTIST MANAGEMENT:
  Add artist:           Create JSON in data/artists/
  Edit artist:          Edit JSON file directly
  View artist:          Use Command Center option [1]

LOGS & MONITORING:
  View activity log:    type data\autopilot_actions.log
  View briefings:       dir data\briefings
  Check queue:          type data\autopilot_queue.json

11. TROUBLESHOOTING
================================================================================

ISSUE: Autopilot not queueing check-ins
FIX: 
- Check health_threshold in config
- Verify days_since_contact_threshold
- Look for errors in autopilot_actions.log
- Ensure auto_checkins.enabled = true

ISSUE: Email not sending
FIX:
- Verify .env credentials
- Check rate limits (may be maxed out)
- Test with: python scripts\maestro_autopilot.py briefing
- Check Gmail "Less secure app access" settings

ISSUE: Artist health always 0/100
FIX:
- Check last_contact_date exists in JSON
- Verify communication_history exists
- Run health calculation manually in Command Center

ISSUE: BRIDGE Intelligence errors
FIX:
- Check ANTHROPIC_API_KEY in .env
- Verify internet connection
- Look for API rate limits
- Check autopilot_actions.log for errors

ISSUE: Queue stuck on "PENDING_APPROVAL"
FIX:
- Open queue: python scripts\maestro_autopilot.py queue
- Approve or reject manually
- Or clear queue: del data\autopilot_queue.json

ISSUE: UnicodeEncodeError
FIX: Already fixed! All file writes use encoding='utf-8'
