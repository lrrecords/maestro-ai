1 — Project Overview

MAESTRO AI is an agentic orchestration system built for LR Records. It routes CLI commands to specialist AI agents, each responsible for a distinct area of label operations. All agents are powered by the Anthropic Claude API and operate on structured artist profile data stored as JSON files.

The system is intentionally modular — new agents are added to scripts/agents/ and registered in maestro.py without touching existing agents. The orchestrator handles artist discovery, command routing, output saving, and the unified full run.

2 — Agents

🎵

Live

Release checklist generator. Takes an artist profile and produces a structured, phased checklist covering pre-release, release day, and post-release tasks. Output saved to data/vinyl/ as JSON.

📣

Live

2-week social media content planner. Generates platform-specific posts accounting for artist voice, genre, and release timing. Output saved to data/echo/ as JSON.

📊
ATLAS
Live

Analytics agent. Ingests sales CSVs from Labelgrid, Orchard, and Bandcamp, normalizes to a common schema, runs Python-side trend calculations, then passes a structured summary to Claude for strategic insights. Output saved to data/atlas/.

SAGE
Planned — Next

Weekly priority planner and PA layer. Will ingest the most recent outputs from VINYL, ECHO, and ATLAS and synthesise them into a ranked weekly action list — one focused brief per artist, with blocked items flagged for human decision.

4 — ATLAS: Data Sources & Schema

ATLAS normalizes three different CSV formats into a single internal schema before any calculations or Claude prompting occurs. All revenue figures are treated as USD; Bandcamp non-USD rows are included at face value (see limitations).

SourceRevenue fieldDate fieldNotes
Labelgridsubtotal_usddsp_transaction_dateMay contain blank territory rows (YouTube-CIDAR)
OrchardRoyaltyDate/TimeGenerally clean
Bandcampnet amountdateMulti-currency — non-USD at face value

What ATLAS calculates in Python (before Claude)

  • Total revenue per source
  • Monthly revenue breakdown across all sources
  • Top performing tracks by revenue
  • Month-on-month trend percentage
  • Record count per source (used in full run summary)

What Claude contributes

Strategic interpretation — anomaly flags, release impact analysis, platform health assessment, and prioritised recommendations. Claude receives the pre-calculated summary only, not raw CSV data.

Each artist is stored as a JSON file in data/artists/. The orchestrator matches by filename slug or the artist_info.name field. Key sections expected by agents:

6 — Known Limitations

⚠️ Bandcamp multi-currency
Non-USD revenue (GBP, EUR, AUD) is included at face value. ATLAS logs a warning but does not apply FX conversion. Any cross-source total revenue figure is approximate when Bandcamp non-USD rows are present.
📋 Blank territory rows — Labelgrid
YouTube-CIDAR exports often contain rows with no territory value. ATLAS flags these but does not exclude or auto-resolve them. They are included in revenue totals.
📋 No data recency concept
ATLAS has no understanding of expected export recency. A -70% month-on-month drop could mean catalog decline or simply that recent data hasn't been exported yet. Both scenarios look identical to the agent.
📋 Deferred items
Three improvements discussed but not yet implemented: VINYL mode switching (single/EP/album modes), unified built_at_utc timestamp field across all agent outputs, and a manifest writer to track all generated files per run.

9 — Next Session: SAGE Agent

SAGE is the fourth and final core agent. It acts as the weekly PA layer, pulling the most recent outputs from VINYL, ECHO, and ATLAS and synthesising them into a single prioritised action list for the working week.

Planned inputs

  • Most recent VINYL checklist — what release tasks are outstanding
  • Most recent ECHO plan — which posts are due this week
  • Most recent ATLAS report — any revenue anomalies or urgent flags

Planned output

  • Ranked weekly action list with urgency tags (NOW / THIS WEEK / MONITOR)
  • One-paragraph weekly brief per active artist
  • Blocked items surfaced for human decision

CLI commands to add

python scripts\maestro.py sage "Artist Name"   # single artist
python scripts\maestro.py sage --all           # full roster
