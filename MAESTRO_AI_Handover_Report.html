Executive Summary

Maestro AI orchestrates artist relationship health, release planning, content strategy, analytics, and automation from a single CLI. The system is fully operational, with agents integrated and a color-coded dashboard for rapid status review.

Entry point
scripts/maestro.py
Key commands
bridge • sage • report • full
Data root
data/

Quick Start

  1. Create .env in project root with ANTHROPIC_API_KEY
  2. Activate venv and install: pip install -r requirements.txt
  3. Run a health snapshot: python scripts\maestro.py bridge "Artist Name"
  4. See the dashboard: python scripts\maestro.py report
Optional: Set CHECKIN_WEBHOOK_URL to notify n8n on health updates.

Agent Overview

BRIDGE — Relationship health scoring and snapshot
  • Outputs: data/bridge/<slug>_YYYYMMDD_HHMMSS.json
  • Fields include: health.score, health.status, pattern_analysis.trend, days_since_contact
  • Triggers optional webhook on completion (n8n integration)
SAGE — Weekly prioritized action plan
  • Integrates outputs across agents to create a Monday-ready plan
  • Roster mode: sage --all
VINYL — Release checklist by phase
  • Phases: pre-release, launch, post-release
  • Shows tasks, owners, and dependencies
ECHO — 14-day content plan
  • Platform-specific copy aligned to artist voice
  • Calendar-ready posts with themes and CTAs
ATLAS — Analytics from CSVs
  • Reads CSVs under data/metrics/
  • Highlights anomalies, trend lines, growth signals
FORGE — Automation specifications
  • Produces n8n-ready specs for operational flows
  • Ideal for notifications, reminders, and sync tasks
CHECKIN — Contact logging
  • Updates artist contact history to inform BRIDGE
  • Optional webhook to alert operations channel

Health Dashboard

Run report for a color-coded overview. Banding:

  • Good (70–100) — Healthy relationship
  • Monitor (40–69) — Watch carefully
  • Critical (0–39) — Take action now
  • No Data — Run BRIDGE to generate a first snapshot

The dashboard reads the most recent file per artist from data/bridge/.

Webhook / n8n Integration

BRIDGE can POST a health update after each run. Import the included health_alert_workflow.json into n8n and ensure the endpoint matches your environment.

Default endpoint

http://localhost:5678/webhook/health-update

Example payload

{
  "artist": "Jerry Mane",
  "score": 14,
  "status": "Critical",
  "trend": "stable",
  "timestamp": "2026-03-07T02:30:00Z",
  "slug": "jerry_mane"
}

If n8n is not running, Maestro will skip the webhook and continue. This is expected and safe.

Adding an Artist

  1. Create data/artists/<slug>.json with at least:
{
  "artist_info": {
    "name": "Artist Name",
    "email": "artist@example.com"
  },
  "musical_identity": {
    "primary_genre": "Genre"
  },
  "upcoming_release": {
    "title": "Release Title"
  },
  "communication_history": []
}
  1. Run python scripts\maestro.py bridge "Artist Name" to generate the first snapshot.
  2. Check report to confirm the score appears.

Troubleshooting

Webhook connection refused

Ensure n8n is running and the URL is correct. Otherwise set CHECKIN_WEBHOOK_URL empty to disable. The system still completes without webhooks.

Artist not found

Run python scripts\maestro.py artists to list available names, or create a profile JSON under data/artists/.

Git push DNS error

Network hiccup. Try ipconfig /flushdns, disable VPN, or retry later.
